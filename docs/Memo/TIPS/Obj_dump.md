# 同名ファイルの出力におけるObj衝突の解消  
時々これどうするんだっけ？となるのでメモ.  
例えば`A/foo.cpp`と`B/foo.cpp`というファイル構成があったとする.  
この際Visual Studioの出力がデフォルトのままだと、Aの`foo.obj`とBの`foo.obj`が同じフォルダに出力されることになる.  
こうなると先にビルドした方が消えて後にビルドされた方が採用されることになる.  
更にWarningも出るので、とりあえず直したいところ.  
この解決は[Visual Studioで単一のプロジェクト内に同名のソース・ファイルが含まれていてもビルドが通るようにする方法](https://altebute.hatenablog.com/entry/2015/01/01/235301)が非常に参考になる.  
ずばり、  

1. プロジェクト->プロパティ->構成プロパティ->C/C++->出力ファイルを開く
2. `ASMリストの場所`と`オブジェクトファイル名`をデフォルトの`$(IntDir)`から`$(IntDir)%(RelativeDir)`とする

こうすればいい感じになる.  
つまり、`A/foo.obj`と`B/foo.obj`の二つが出力されて衝突を免れることになる.  
デフォルトの衝突を直したいときはこれをやっておくとよい.
